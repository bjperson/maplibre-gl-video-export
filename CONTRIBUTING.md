# Contributing

Thanks for your interest!

## Before Contributing

- **Open an issue first** to discuss your idea
- Make sure it aligns with the project goals
- Check if someone is already working on it

## Pull Request Guidelines

- **Squash commits** - One logical change per PR
- **Write tests** - Maintain code coverage
- **Update docs** - Keep README/docs in sync
- **Follow existing style** - Match the codebase patterns

## Commit Message Format

```
type: Brief description

Longer explanation if needed

```

Types: `feat`, `fix`, `docs`, `refactor`, `perf`, `test`

## What We Accept

- ✅ Bug fixes with tests
- ✅ Performance improvements with benchmarks
- ✅ New animations (high quality only)
- ✅ Documentation improvements

## What We Don't Accept

- ❌ Typo-only PRs (we'll batch these ourselves)
- ❌ Whitespace/formatting-only changes
- ❌ Dependency bumps without reason

---

*Please be patient - reviews may take a few days.*
